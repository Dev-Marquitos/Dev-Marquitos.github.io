01 — Fundamento Base

¿Qué es la Programación?

La programación es el arte y la ciencia de escribir instrucciones precisas para que una computadora ejecute tareas específicas. Es el lenguaje mediante el cual los humanos se comunican con las máquinas.

Definición

Proceso de diseñar y crear código fuente que una computadora puede ejecutar para resolver problemas.

Objetivo

Automatizar tareas, procesar información y crear soluciones eficientes a problemas complejos del mundo real.

Proceso

Análisis → Diseño → Codificación → Pruebas → Mantenimiento: el ciclo de vida completo de un programa.

Importancia

Base de toda la tecnología moderna: apps móviles, inteligencia artificial, sistemas operativos, videojuegos y más.

02 — Lógica y Proceso

¿Qué es un Algoritmo?

Un algoritmo es una secuencia finita, ordenada y precisa de instrucciones que resuelven un problema o realizan una tarea. Son la base del pensamiento computacional.

01

Entrada (Input)

El algoritmo recibe datos iniciales que serán procesados. Pueden ser cero o más valores de entrada.

02

Proceso

Una serie de pasos bien definidos transforman los datos de entrada según las reglas establecidas.

03

Decisión

Puntos de bifurcación donde el flujo puede cambiar según condiciones evaluadas en tiempo de ejecución.

04

Salida (Output)

Resultado final del proceso: el algoritmo produce uno o más valores como solución al problema planteado.

// Diagrama de Flujo

▶ INICIO
Leer datos de entrada
Procesar información
¿Válido?
Mostrar resultado
■ FIN

Propiedades de un buen algoritmo

✓ Finito: termina después de un número finito de pasos
✓ Definido: cada paso está bien definido y es preciso
✓ Eficiente: usa los recursos mínimos necesarios
✓ Correcto: produce el resultado esperado siempre

04 — Flujo del Programa

Estructuras de Control

Las estructuras de control determinan el flujo de ejecución de un programa. Son los mecanismos que permiten tomar decisiones, repetir acciones y organizar el código.

Estructuras Condicionales

Permiten ejecutar bloques de código diferentes según el resultado de una condición booleana. Son la base de la toma de decisiones en programación.

if (nota >= 70):
    print("Aprobado ✓")
elif (nota >= 50):
    print("Recuperatorio")
else:
    print("Reprobado ✗")
if else if else switch

Estructuras Iterativas

Los bucles permiten repetir un bloque de código múltiples veces. Esenciales para procesar colecciones de datos y realizar cálculos repetitivos.

# Bucle for
for i in range(1, 6):
    print(f"Iteración {i}")

# Bucle while
contador = 0
while contador < 5:
    contador += 1
for while do-while foreach

Instrucciones de Salto

Alteran el flujo normal de ejecución, permitiendo salir de bucles, continuar con la siguiente iteración o retornar valores de funciones.

for i in range(10):
    if i == 3: continue # Salta
    if i == 7: break   # Sale
    print(i)

def funcion():
    return "resultado"

06 — Estilos de Programar

Paradigmas de Programación

Un paradigma de programación es un estilo o filosofía para resolver problemas mediante código. Define cómo se estructura, organiza y ejecuta el programa.

01
🏗️
Orientado a Objetos

Organiza el código en objetos que combinan datos (atributos) y comportamientos (métodos). Basado en los principios de encapsulamiento, herencia, polimorfismo y abstracción.

Java Python C++ C#
02
⚗️
Funcional

Trata la computación como la evaluación de funciones matemáticas puras. Evita el estado mutable y los efectos secundarios. Ideal para procesamiento de datos y concurrencia.

Haskell Scala F# Erlang
03
📋
Imperativo / Procedural

Describe la computación como una secuencia de instrucciones que modifican el estado del programa. El más intuitivo: describe paso a paso cómo lograr el resultado.

C Pascal BASIC Fortran
04
🧠
Lógico

Basado en la lógica formal. El programador declara hechos y reglas; el motor de inferencia deduce conclusiones. Poderoso para IA, pruebas formales y bases de datos.

Prolog Datalog Mercury

08 — Código de Calidad

Buenas Prácticas

Escribir código que funciona es el mínimo. Un buen programador escribe código limpio, legible, mantenible y eficiente que otros puedan entender y mejorar.

01

Nombres descriptivos y semánticos

Usa nombres que comuniquen intención. calcular_promedio_notas() es infinitamente mejor que fn1(). El código se lee más veces de las que se escribe.

02

Principio DRY — Don't Repeat Yourself

Si escribes el mismo código más de una vez, crea una función. La duplicación lleva a inconsistencias y dificulta el mantenimiento a largo plazo del sistema.

03

Comentarios útiles y documentación

Comenta el por qué, no el qué. El código debe ser auto-explicativo; los comentarios explican decisiones de diseño y lógica compleja.

04

Funciones pequeñas y con única responsabilidad

Cada función debe hacer una sola cosa y hacerla bien (Principio SRP). Si tu función tiene más de 20 líneas, probablemente hace demasiado.

05

Control de versiones desde el inicio

Usa Git desde el primer commit. Un historial claro de cambios es invaluable para entender la evolución del proyecto y recuperarse de errores.

06

Testing: prueba antes de que fallen

Un código no probado es código roto esperando a ser descubierto. Las pruebas unitarias y de integración son inversiones en confianza y calidad.
